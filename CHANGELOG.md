# 更新日志

本项目从 `Alpha 0.0.1` 开始采用语义化版本。项目配置中的等价版本号为
`0.0.1-alpha.0`：

- 主版本号：不兼容的结构调整、数据迁移或重大产品升级。
- 次版本号：新增功能或完整功能板块。
- 修订号：问题修复、文案调整和不改变使用方式的体验优化。

每次发布都需要同步更新 `package.json`、`package-lock.json` 和
`lib/changelog.ts`，确保网页展示的版本与项目版本一致。

Alpha 阶段用于验证核心功能和使用体验；功能稳定后再进入 Beta 和正式版本。

## Alpha 0.0.4 — 2026-07-20

- 今日工作内容支持四种推进状态，长文本会自动换行并展开。
- 昨日复盘增加状态更新、返回编辑和未完成事项加入今天等交互。
- 明日计划增加用户可配置的邮件提醒设置，以及 Supabase Cron、Edge Function 和 Resend 服务端提醒链路。
- 周期复盘填写区支持展开和收起。
- 全站增加更圆润的卡片、输入框和按钮，并提高重点文字字号与交互反馈。
- 正式生产域名更新为 `https://www.daily-work-review.com/`。

## Alpha 0.0.3 — 2026-07-17

- 留言反馈迁移至独立子页面，主页恢复紧凑的信息密度。
- 主页页脚增加轻量留言反馈入口。
- 独立反馈页面沿用现有登录状态、视觉风格和移动端适配。

## Alpha 0.0.2 — 2026-07-17

- 新增留言反馈板块，可提交功能建议、问题反馈、体验优化和其他留言。
- 留言通过邮件服务转发到开发者邮箱，并自动附带当前版本信息。
- 自动填入登录账号邮箱作为可选联系方式。
- 增加发送状态、错误反馈和防垃圾留言蜜罐字段。

## Alpha 0.0.1 — 2026-07-17

日迹首个公开体验版本，包含：

- 每日工作复盘与明日计划。
- 周、月、季度和年度工作总结。
- 待办状态管理与未完成任务自动顺延。
- Supabase 登录注册与云端同步。
- 随机语录、自定义语录和总结复制。
- 全新的登录注册界面、确认密码与交互反馈。
- 桌面端和移动端响应式布局。
